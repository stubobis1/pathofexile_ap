What is Archipelago?

Archipelago is a free, open-source multiworld randomizer. It shuffles the items of one game into the world of another — so when you beat a boss in Path of Exile, you might unlock a power-up for your friend playing Super Mario 64, and vice versa.

You can also play completely solo: your Path of Exile progression gets shuffled with itself, so you earn your skills, gems, and gear slots as rewards instead of starting with them.

🌍
Multiworld
Multiple players in multiple games, all sharing one item pool. Help each other progress.
🎯
Checks
When you find a qualifying item or level up, you "check" a location and send an item to someone.
🔓
Unlocks
Items you receive from others (or yourself) let you equip new gear, use new gems, and more.
⚔️
Goals
Your run ends when you hit your goal — finish the campaign, kill specific bosses, or anything in between.
Is this against Path of Exile's ToS?

No. The client only reads your Client.txt log file and the public PoE API, then writes to your local item filter. It uses one-button-one-action macros and nothing else. Everything it does is something a regular player could feasibly do manually.

How It Works in PoE

The Archipelago client runs alongside Path of Exile. It never touches the game's files directly — it watches what you do and reacts:

1

You pick up an item or level up.

The client watches your inventory when you change zones. Qualifying items (specific base types) count as "checks" and send items to other players.

2

Your item filter highlights what matters.

The client modifies your local loot filter to highlight items that would send a check, and stops highlighting items you've already checked. Optional audio cues announce incoming unlocks.

3

You receive unlocks from the multiworld.

When you (or a teammate) checks a location, you might receive a skill gem, a gear slot upgrade, passive points, or a new character class. The client tells you what arrived.

4

You can only use what you've unlocked.

Equipping something you haven't unlocked puts you "out of logic" and blocks further check sends until you correct it. Whisper yourself !gear or !gems to see what's allowed.

Important

Checks are only sent when you change zones. If you pick something up and nothing happens right away, don't panic — just walk through a waypoint or door.

New to Path of Exile?

In a normal PoE run you pick up any gem and equip any item you find. Here, you start with almost nothing — you'll earn the right to use gems, wear better gear, and spend passive points as rewards for exploring the world. Think of it like starting a run with a locked skill tree that you gradually unlock by playing.

Before You Begin

✓

Path of Exile — installed and playable. Free download here.

✓

Python 3.12 — required by Archipelago. Download Python 3.12.

Python 3.13 will NOT work. Use 3.12 specifically.

✓

Know the path to your PoE Client.txt log file — usually somewhere like C:\Program Files (x86)\Grinding Gear Games\Path of Exile\logs\Client.txt

✓

Have a local item filter file saved on your PC — not a subscribed one. Usually in C:\Users\YOU\Documents\My Games\Path of Exile\. Any NeverSink variant works fine as a base.

Step 1 · Install Archipelago

1

Download the latest Archipelago installer from github.com/ArchipelagoMW/Archipelago/releases.

Get the .exe installer, not the source code archives.

2

Run the installer and choose a convenient folder (e.g. C:\Games\Archipelago).

3

Launch ArchipelagoLauncher.exe from that folder to confirm it works. Consider making a desktop shortcut — you'll use this often.

Step 2 · Add the Path of Exile .apworld

1

Download the latest poe.apworld file from github.com/stubobis1/Archipelago/releases.

Again, grab the .apworld file — not the source code.

2

Copy that file into the custom_worlds folder inside your Archipelago installation. Do not extract or rename it — just drop the file in.

Step 3 · Generate or Join a Game

You need a .yaml settings file that tells Archipelago what kind of run you want.

Creating your settings file

1

In ArchipelagoLauncher.exe, click Generate Template Options (near the bottom of the list).

2

Open Archipelago/Players/Templates/Path of Exile.yaml in any text editor.

3

Edit to taste. Key settings:

  • name — your player slot name (e.g. ChrisWilsonsBeard)
  • goal — what ends your run (campaign, bosses, etc.)
  • starting_character — which class you begin with (or random)
  • usable_starting_gear — how much starter gear you keep
  • add_passive_skill_points_to_item_pool — locks and unlocks passive points
4

Save your .yaml to the Archipelago/Players/ folder (not Templates).

Tip — Easy Options Editor

You can also build your yaml visually at ap.stripesoo7.org or multiworld.gg.

If you are hosting (solo or with friends)

1

Make sure all players' .yaml files are in the Players/ folder.

2

Click Generate in the Launcher. This creates an archive in the output/ folder.

3

Click Host in the Launcher and select that archive. The server runs on port 38281 by default. Share your IP and port with teammates.

If someone else is hosting

Send the host your .yaml file and make sure they have the same poe.apworld version. Get the server address and your slot name from them.

Step 4 · PopTracker (Optional but Recommended)

PopTracker gives you a visual dashboard of what gems, gear slots, and ascendancies you've unlocked.

1

Download PopTracker from github.com/black-sliver/PopTracker/releases and extract it anywhere.

2

Download the PoE pack (a .zip archive) from github.com/stubobis1/PathOfExilePoptracker/releases.

3

Drop that zip (without extracting it) into poptracker/packs/.

4

Launch PopTracker.exe, click the AP button at the top, and enter your server address and slot name. The button turns green when connected.

Step 5 · Start the Client & Play

1

In ArchipelagoLauncher.exe, find and click Path of Exile in the client list to open the PoE client window.

2

In the Server bar at the top, enter your address and click Connect. Format: 127.0.0.1:38281 for local, or whatever the host gave you.

When prompted, enter your slot name. If a password is set, use SlotName:password@host:port.

3

Authenticate your PoE account by running /poe_auth in the client console. A browser window will open — click Authorize. This only shares your public profile data.

4

Launch Path of Exile and log in. Note your character's exact name.

5

Back in the client, set your character: /char YourCharacterName

6

Set your log path: /client "C:\...\logs\Client.txt" (include the quotes)

7

Set your item filter: /filter MyFilter.filter
This should be a local filter file in your Path of Exile documents folder.

8

Type /start and enter the game. The item filter will load and you'll see highlighted items immediately.

Enter a zone to trigger your first check. Welcome to the randomizer!

In-Game Chat Commands

Whisper these commands to your own character in PoE chat. For example: @YourCharName !gems

!gemsShow all gems received
!skill gemsList usable skill gems by level
!support gemsList support gems received
!utility gemsList utility gems (auras, curses…)
!gearShow what gear you can equip
!weaponsShow usable weapons
!armorShow usable armour
!linksShow your maximum allowed linked sockets
!flasksShow flask slot unlocks
!passiveList usable passive skill points
!ascendancyShow unlocked ascendancies
!goalView your current run goal
!deathlinkToggle DeathLink (share deaths with others)
!helpShow all commands

Client console commands (typed in the Archipelago client, not in PoE):

/receivedList all items you've received
/poe_authAuthenticate with PoE (run once)
/char NameSet the character to track
/client "path"Set Client.txt path (quotes required)
/filter name.filterSet your local item filter
/startStart the randomizer session
/deathlinkToggle DeathLink
Keyboard Shortcuts

F11 — restart the client if something goes wrong.
F12 — force the client to re-check your current state (useful if a check didn't send).

Tips & Troubleshooting

  • Keep the Archipelago client open the entire time you play. It shows item sends, receives, and player names in real time.
  • Checks only fire on zone change. Nothing happening after a pickup is normal — just move to a new area.
  • If you're using a gem or gear piece you haven't unlocked, you're "out of logic" and checks won't send until you fix it. Whisper yourself !gear or !gems to verify what's allowed.
  • If your item filter isn't updating, check that the path is correct and that OneDrive sync isn't locking the file. Try disabling OneDrive if you see write errors.
  • Make sure the /client path points to the correct Client.txt. The client reads this file to detect zone changes and chat whispers.
  • Town Portals are your friend — drop items off at the stash to check them without needing to reach a full transition zone.
  • The early levels before your first few gems arrive can be rough. There's no shame in starting with your class weapon, starter gem, and flasks unlocked in your yaml settings.
  • Many "bad" endgame skills are perfectly fine for clearing the campaign. Use whatever you've been given!
  • F12 forces a re-check if you think something got missed — helpful when Client.txt is temporarily unreadable.
  • Report bugs in the GitHub issue tracker or ask in the Path of Exile Archipelago Discord channel.
